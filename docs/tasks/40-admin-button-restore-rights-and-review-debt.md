внешние блокеры v1 (ЮKassa, юрпакет) не сняты.

## Задача 2: Восстановление магазина — право по актёру удаления

Пересматривает [Спринт 37](37-seller-soft-delete-and-monitoring-retry.md)
— «восстановить может продавец или админ» верно только когда удалил
себя сам продавец; если удалил/заблокировал админ — восстановить может
только админ.

- [x] Миграция: поле `deletedBy: 'seller' | 'admin'` в `sellers`
  (`drizzle/0005_dusty_energizer.sql`, enum `seller_deleted_by`). Единый
  источник значений — `sellerDeletedByActors` в `packages/shared`, тот же
  паттерн, что у `sellerStatuses`.
- [x] `seller.service.ts` (`deleteSeller`) — пишет `deletedBy: 'seller'`.
- [x] `platform.service.ts` (`updateSellerStatus`) — пишет
  `deletedBy: 'admin'`.
- [x] `apps/api/src/routes/seller.route.ts` (`POST /seller/restore`) —
  проверка актёра: 403, если `deletedBy === 'admin'` и вызывает не
  админ. Найден и обработан краевой случай, не описанный в исходной
  формулировке: владелец платформы — одновременно продавец своего
  демо-магазина и админ (см. `project_owner_telegram_id` в памяти) — если
  такая сессия удалит **свой же** магазин через админский путь и попробует
  восстановить его через продавцовский `/seller/restore`, `isAdmin` из

## Задача 4: Кнопка «Удалить» вылезает за карточку в админке платформы

Заведено 23.07.2026 по прямому запросу пользователя. Строка действий
продавца в `PlatformHome.tsx:147-202` (`div.flex.items-center.
justify-between`, без `flex-wrap`) — слева число месяцев + «Выдать
доступ», справа «Заблокировать»/«Удалить» (`text-tg-destructive`).
В отличие от строки заголовка карточки чуть выше (`:119`, там явно
`min-w-0 flex-1` на тексте и `shrink-0` на бейдже статуса), в этой
строке ни один из блоков не имеет `shrink-0`/переноса — на узком экране
суммарная ширина инпута и трёх кнопок превышает ширину карточки, и
`justify-between` без `flex-wrap` выталкивает правую группу (в т.ч.
«Удалить») за пределы карточки, а не переносит на новую строку.

- [x] `PlatformHome.tsx:147` — `flex-wrap` на строке действий: правая
  группа кнопок теперь переносится на новую строку, а не выталкивается
  за карточку.
- [x] Узкие вьюпорты — то же ограничение среды, что и в Спринте 34/33:
  Playwright не установлен в песочнице (`Cannot find module 'playwright'`
  при попытке запустить Chromium), проверено разбором классов, не
  реальным рендером. Стоит перепроверить глазами при следующей
  возможности (`pnpm dev`).
