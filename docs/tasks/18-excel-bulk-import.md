**Инцидент при деплое, найден и исправлен 20.07.2026**: после пуша
`git push origin main` панель импорта не появилась на проде, хотя CI
(`test→build→deploy`) отчитался зелёным. Причина — гонка между двумя
CI-ранами: коммит `2a117df` (только доки, «Спринт 18 согласован») и
следом `7f20f40` (код) породили два параллельных рана; оба
`build-and-push` пушат один и тот же мутируемый тег `:latest`, ран
`2a117df` финишировал на ~2 секунды позже и затёр `:latest` версией
без кода Спринта 18 — `deploy` уже нового коммита стянул устаревший
образ. Диагностировано SSH на VM: `git log` на сервере показывал
верный коммит, но собранный JS-бандл не содержал строк фичи; локальная
пересборка того же Dockerfile с тем же кодом дала бандл с фичей —
разница была именно в том, какой образ подхватил `:latest`, SHA-теги в
GHCR были корректны у обоих. Восстановление — на VM образы
перетегированы с точного SHA `7f20f40...` обратно в `latest`,
`docker compose up -d`, проверено внешним `curl`: бандл содержит
`products/import`, `products-template.xlsx` отдаётся 200. Профилактика —
`concurrency`-группа в `.github/workflows/ci.yml` (`cancel-in-progress:
true` по ветке, см.
[STACK.md#ci-cd](STACK.md#ci-cd)) — более старый ран теперь отменяется
целиком при новом пуше, до registry доезжает только последний коммит.
Фикс проверен собственным пушем (коммит `3fca5ab`) — CI прошёл
`test→build→deploy` естественным путём, прод отдал тот же корректный
бандл без ручного вмешательства.

**Спринт 18 закрыт** — продавец скачивает шаблон, заливает каталог
файлом `.xlsx`, валидные карточки создаются, ошибки по невалидным
строкам показаны построчно; попутно найден и исправлен инцидент гонки
за тегом `:latest` в CI/CD.
