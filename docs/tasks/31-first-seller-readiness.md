### Спринт 31: подготовка к первому живому продавцу

Цель: покупательский путь чекаута устойчив к сетевым сбоям и гонке остатка
без риска задвоить заказ, сообщения бота продавцу/покупателю оформлены не
как голый служебный текст, у платформы есть растровый favicon на обеих
публичных поверхностях.

Состав согласован в диалоге 22.07.2026 (`/grill-me`). P0-блокеры v1 (ИП,
онбординг ЮKassa, юрпакет) — вне кода и вне этого спринта, идут своим
чередом параллельно. Юнит-экономика v1 — по теме, но не код, оставлена вне
спринта, отдельный заход.

Задачи:

- [ ] Общая экспортируемая строка-константа в `packages/shared` для ошибки
  «нет остатка» — API (`orders.route.ts`/`orders.service.ts`) и фронт
  (`CheckoutPage.tsx`) сравнивают с одной и той же константой вместо
  дублирования литерала. Не заводить новое поле `code` в теле ошибки —
  во всём API сейчас единственный контракт `{ error: string }`, менять
  его ради одного эндпоинта не стоит (см. диалог 22.07.2026).
- [ ] Отдельное сообщение в `CheckoutPage.tsx` для сетевой/офлайн ошибки
  создания заказа, отличное от сообщения о нехватке остатка (сейчас оба
  случая заходят в один общий `createOrder.isError`).
- [ ] Idempotency-key на создание заказа: клиент генерирует UUID один раз
  на попытку оформления (не на каждый повтор), сервер дедуплицирует —
  новая колонка в `orders` + уникальный индекс, миграция, TDD. Закрывает
  риск дубля заказа, если пользователь нажимает «Оформить заказ» повторно
  после сетевой ошибки, а исходный запрос на самом деле дошёл до сервера.
- [ ] Оформление текста сообщений бота — Markdown-разметка, эмодзи, единый
  тон: `start-handler.ts` (уже есть inline-кнопка, меняется только текст)
  и `order-notification.ts` (`formatOrderNotificationText` — сейчас голый
  текст без форматирования). Кнопка-диплинк на конкретный заказ — вне
  скоупа (требует нового формата `start_param` и роутинга по роли на
  фронте), заведена отдельным пунктом в [backlog.md](backlog.md).
- [ ] Растровый favicon из `docs/design/logo.svg` — сгенерировать `.ico` +
  набор PNG через `sharp` (уже зависимость `apps/api`, ImageMagick/Inkscape
  в окружении нет), подключить на обеих публичных поверхностях:
  `apps/web/index.html` (ТМА) и `apps/web/public/landing.html`. Аватарка
  `@grammashopbot` в BotFather — ручное действие пользователя, не входит
  в задачу.
- [ ] `CheckoutPage.tsx` — перевести `border-emerald-500` на токен
  `--tg-success` (хвост Спринта 29, находка 1, см.
  [tasks/29-admin-design-review.md](29-admin-design-review.md)) — без
  визуальных изменений, то же значение цвета.
- [ ] Подсказка в `db:seed` про видимость seed-продавца (протухший
  `paid_until` лечится перезапуском скрипта) — зафиксировать это прямо в
  выводе/комментарии скрипта.

Реальная работа (аудит, код) не начинается без отдельной команды старта —
состав зафиксирован в диалоге 22.07.2026.
