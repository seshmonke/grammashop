### Спринт 21: регистрация магазина до оплаты + льготы

Цель: любой пользователь Telegram может сам завести магазин и наполнить
каталог без оплаты; витрина открывается покупателям только при активной
подписке, которую до готовности ЮKassa админ выдаёт вручную (льгота =
сдвиг `paid_until`). Состав согласован `/grill-me` 20.07.2026, решения
зафиксированы в
[CONCEPT.md#оплата-подписки-продавцом](../CONCEPT.md#оплата-подписки-продавцом)
(регистрация отделена от оплаты, обязательные поля, механика льготы,
исправление ошибочного предположения о рекурренте ЮKassa). Реализует
незаблокированную часть п.3 продуктовой карты (форма регистрации без
ЮKassa-интеграции); заодно закрывает пункты очереди «Скрытие витрины при
suspended», «Редирект на лендинг для пользователя без магазина», «Ссылка
на витрину в админке», «+7 в чекауте». Задачи идут в порядке
зависимости.

**Анализ перед стартом** (20.07.2026, по правилу из CLAUDE.md; ресёрч —
документация Telegram Mini Apps и ЮKassa):

- Форма регистрации: у Mini Apps есть нативный `requestContact()` с
  подписанными данными — телефон предзаполняется из аккаунта, а не
  вводится руками; встроено в задачу формы. `openLink` не закрывает
  ТМА — экран-развилка безопасен.
- Льгота: вскрыт краевой случай отсчёта от истёкшего `paid_until` —
  правило «от большего из (сейчас, paid_until)» встроено в задачу.
  Совместимость с рекуррентом подтверждена документацией ЮKassa ещё при
  согласовании состава (планировщика у ЮKassa нет, списания инициирует
  платформа).
- Копирование ссылки: запись в буфер (`navigator.clipboard.writeText`)
  в Mini App не ограничена (ограничено только чтение) — кнопка
  «Скопировать» рабочая, fallback — выделяемый текст ссылки.
- CTA лендинга `startapp=register`: добавлен гвард «у пользователя уже
  есть магазин → админка, не форма» — иначе продавец, ткнувший CTA,
  упирался бы в 409.
- Сквозная проверка: выявлена потребность во втором Telegram-аккаунте
  (владелец уже seed-продавец) — помечено в задаче как ручное действие.
- Остальные задачи (диагностика, контракт, CRUD профиля, баннер, кнопка
  админки, карусель, `+7`) — без развилок реализации, ресёрч не
  требовался.
