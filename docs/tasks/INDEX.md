# Задачи

Скрам-доска проекта: очередь → спринт → в работе → выполнено. Когда
задача решена, итоговое решение уходит в чистовик (`CONCEPT.md`/
`STACK.md`), а здесь остаётся короткая запись со ссылкой — этот файл про
статус, не про содержание решений.

Разбито на папку 21.07.2026 (было — один файл `docs/TASKS.md` на 2000+
строк с историей всех спринтов сразу): один файл — один спринт, здесь —
только оглавление и правила. Ссылки на этот файл вида `TASKS.md` в
старых коммитах/комментариях кода теперь ведут сюда;
[backlog.md](backlog.md) и [milestones.md](milestones.md) — бывшие
разделы «Очередь» и «Выполнено».

## Воркфлоу спринтов

Спринт — не отрезок календарного времени, а пронумерованная пачка задач.
Система дат (дд.мм–дд.мм) упразднена 17.07.2026: реальный темп (шесть
«недельных» вех дорожной карты закрыты за два дня) сделал привязку к
неделям фиктивной — доска врала о темпе, а не описывала его. Соло-проект,
поэтому без лишних ролей и церемоний — просто явные границы «что делаем
сейчас».

1. **Бэклог** — [backlog.md](backlog.md). Весь пул задач без привязки к
   времени, не отсортирован по приоритету жёстко.
2. **Планирование** — при открытии нового спринта заводим новый файл
   `docs/tasks/NN-краткое-описание.md`: `### Спринт N: цель` (цель —
   одна фраза, что должно быть верно к завершению спринта), под ней —
   список задач, перенесённых из бэклога (столько, сколько реально
   сделать за один присест, не больше), плюс запись в списке спринтов
   ниже.
3. **В работе** — задача, которую делаем прямо сейчас, внутри файла
   текущего (последнего по номеру) спринта. Не больше 1-2 одновременно,
   чтобы не размазываться.
4. **Сверка** — короткий проход по доске перед началом новой пачки работы
   (не встреча, соло-проект):
   - что сделано → отметить в файле спринта;
   - что берём в работу дальше → обновить внутри того же файла;
   - приоритеты в [backlog.md](backlog.md) → перепроверить по текущим
     блокерам, чтобы метки `P0/P1/P2` не протухали (закрылся внешний
     блокер — что было `P2` за ним, могло стать `P1`);
   - что новое всплыло (в диалоге, в коде, где угодно) → сразу
     завести задачей в [backlog.md](backlog.md), а если это горит и
     укладывается в цель спринта — сразу в файл текущего спринта.
   Смысл сверки — не дать новым задачам потеряться и не копить их «в
   уме»: всё, что всплыло, сразу попадает в файл.
5. **Обзор и ретро** — в конце спринта:
   - сделанное → короткая ссылка на раздел чистовика прямо в файле спринта;
   - несделанное → обратно в [backlog.md](backlog.md) (или явно в
     следующий спринт, если это по-прежнему приоритет);
   - при необходимости — короткая заметка в самом файле спринта: что
     мешало, что стоит изменить в следующем.
6. Файл закрытого спринта остаётся в папке как история (не удаляется и
   не переносится), открывается файл следующего спринта.

Общий обзор вех инженерного трека и продуктовой карты — [roadmap.md](roadmap.md).
Решения концептуального уровня, закрытые попутно со спринтами (не сами
спринты, а выводы) — [milestones.md](milestones.md).
