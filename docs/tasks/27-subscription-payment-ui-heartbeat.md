### Спринт 27: UI оплаты подписки + heartbeat-мониторинг

Цель: продавец может оплатить/продлить подписку прямо из своей админки
(фронт поверх бэкенда Спринта 26), и платформа видит молчаливые отказы
(упавший ночной бэкап, легшая VM, отвалившийся бот) — не только
исключения в Sentry. Два независимых мелких пункта без внешних блокеров
— в отличие от Спринта 26 здесь нет риска размазать присест на одной
крупной задаче, наоборот, каждый пункт мал сам по себе. Состав согласован
в диалоге 21.07.2026, сразу после закрытия Спринта 26 и его довеска —
сквозной проверки движка против реального тестового магазина ЮKassa.

Задачи:

- [ ] UI-кнопка «оплатить подписку» в админке продавца (`apps/web`):
  кнопка дёргает `POST /seller/subscription/pay` и открывает
  `confirmation_url` (в ТМА — `openLink`/redirect), плюс отображение
  статуса подписки (active/grace/suspended, дата до). Тестируется на
  реальном sandbox ЮKassa — ключи тестового магазина уже в `.env`,
  сам платёжный путь подтверждён Спринтом 26.
- [ ] Heartbeat-мониторинг (наблюдаемость «тишины»): внешний пинг
  `/health` каждые N минут (healthchecks.io/UptimeRobot) → алерт при
  недоступности; бэкап-скрипт пингует свой URL после успешной заливки →
  dead-man's switch при отсутствии пинга. Проверить, что `/health` не
  отдаёт ПДн.
