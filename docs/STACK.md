# Стек проекта

Референс-документ. Здесь фиксируем принятые решения по технологиям, чтобы не
держать их в голове и не переспрашивать через месяц, почему выбрали именно
это.

## Frontend
**React + TypeScript + Vite**
Стандартный, самый живой экосистемный выбор. Vite — минимум конфигурации.

## Backend
**Node.js + Fastify + TypeScript**
Вместо Express: нативная валидация схем и типизация из коробки (через Zod),
меньше ручного wiring, выше производительность.

## Database
**PostgreSQL + Drizzle ORM**
Вместо Prisma: не тащит отдельный query-engine бинарник (легче образ,
быстрее холодный старт), query builder прозрачно мапится на SQL — не
прячет от разработчика, что реально уходит в базу, миграции — читаемые
`.sql`-файлы, а не сгенерированная магия.
Минус: меньше комьюнити и туториалов, чем у Prisma — будем разбираться
через документацию Drizzle/Postgres напрямую, если что-то пойдёт не так.

## Валидация
Zod-схемы в общем пакете (`packages/shared`), шарятся между backend-роутами
и frontend-формами — валидация не дублируется.

## Монорепо
pnpm workspaces:
```
apps/web/        — React
apps/api/        — Fastify
packages/shared/ — Zod-схемы, общие типы
```

## Docker
`docker-compose.yml` для локальной разработки (postgres + api + web),
multi-stage `Dockerfile` на каждый сервис для прода.

## Тестирование
Vitest — единый раннер для web и api.

## Принципы проекта
- Минимум зависимостей, ничего "про запас"
- Простая слоистая архитектура (routes → services → db), без DDD/hexagonal
- Усложняем архитектуру только тогда, когда появляется реальная боль,
  оправдывающая это — не заранее
