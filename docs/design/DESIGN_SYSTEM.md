# Дизайн-система grammashop

Чистовик. Зафиксировано в диалоге 19.07.2026 (Спринт 10, см.
[tasks/10-design-system.md](../tasks/10-design-system.md)). Направление — **Y2K** (хром, магента, ice,
жирная геометричная типографика), пришло на смену раннему черновику
«тихий минимализм + brass» (см. «История» в конце файла — тот черновик
отклонён целиком, включая символ на весах).

Это спецификация **токенов и правил**, не отчёт о реализации: код
`apps/web` (см. «Не реализовано» ниже) всё ещё на системе из Спринта 9
(brass/Fraunces). Перевод интерфейса на эту систему — отдельная будущая
задача.

## Принцип: один слой, не замена

Витрина живёт внутри чужого приложения — фон, текст, light/dark
приходят из Telegram покупателя (`themeParams`, см.
[STACK.md#авторизация](../STACK.md#авторизация)). Это уже проверенное
преимущество: магазин ощущается частью Telegram, а не отдельным вебвью.
Дизайн-система не спорит с этим фактом, а работает **поверх** него:
акцентный цвет, вордмарк, символ, ритм типографики — то немногое, что
принадлежит нам, а не клиенту покупателя.

**Архитектура акцента и тарифная сетка** (см.
[CONCEPT.md#тарифы](../CONCEPT.md#тарифы)): с Тарифа 2 продавец получает
свой цвет и толщину линий, с Тарифа 3 — свой фон. Значит:

- **Витрина продавца** — токены ниже это дефолт Тарифа 1 и стартовое
  состояние Тарифа 2/3 до кастомизации продавцом. Не жёстко зашитый цвет
  платформы.
- **Продавцовская и платформенная админки, аватарка бота, будущий
  лендинг** — токены неизменны. Это айдентика grammashop, не продавца, и
  кастомизации продавцом не подлежит.

## Палитра

| Токен | Hex | Роль |
|---|---|---|
| `--void` | `#0A0A0C` | глянцевый чёрный, база тёмных поверхностей бренда |
| `--void-2` | `#131317` | карточки/панели поверх `--void` |
| `--void-3` | `#1B1B21` | ещё один уровень поверхности (глубже вложенные блоки) |
| `--chrome` | `#B8C4D0` | материал: заголовки, обводки, структура |
| `--chrome-light` | `#F0F4F8` | верхний блик хром-градиента |
| `--chrome-dark` | `#5A6572` | нижняя тень хром-градиента, приглушённый текст на брендовых поверхностях |
| `--magenta` | `#FF2E9A` | акцент — **коммерция/ценность**: CTA, цены, скидки |
| `--magenta-deep` | `#C4157A` | magenta, калибровка под светлые поверхности (не «слепнет» на паппире/белом) |
| `--ice` | `#6EE7FF` | акцент — **интерактив/статус/навигация**: ссылки, фокус с клавиатуры, номера, статус-точки, состояния ожидания |
| `--text` | `#F0F0F2` | основной текст на тёмных брендовых поверхностях |
| `--text-dim` | `#8A8F98` | приглушённый текст |
| `--rule` | `rgba(184, 196, 208, 0.22)` | тонкие разделители |

**Смысловое разделение двух акцентов — не декоративное дублирование:**
magenta несёт коммерческую информацию (что стоит денег/приносит
действие), ice — интерактивную/статусную (что кликабельно, в фокусе,
ожидает ответа). Правило при добавлении новых элементов: сначала
определить, к какой из двух категорий относится элемент, потом красить.

Rationale выбора: chrome (не золото/brass) — материал-сигнатура
Y2K-эпохи (1999-2003), magenta+ice — классическая Y2K-пара
(cyber-bubblegum + ice-cyber), проверено по актуальным источникам
19.07.2026 (не по памяти) — см. «Источники» ниже.

## Типографика

**[Unbounded](https://fonts.google.com/specimen/Unbounded)** (Google
Fonts) — вариативный шрифт, ось `wght` 200–900 в одном файле,
**с кириллицей** (в отличие от раннего черновика на Fraunces, у которой
кириллицы нет вообще — см. «История»).

```css
@font-face {
  font-family: 'Unbounded';
  src: url('.../unbounded-cyrillic.woff2') format('woff2');
  font-weight: 200 900;
}
@font-face {
  font-family: 'Unbounded';
  src: url('.../unbounded-latin.woff2') format('woff2');
  font-weight: 200 900;
  unicode-range: U+0000-00FF, U+2000-206F;
}
```

Google Fonts CSS2 API отдаёт **один и тот же файл** на разные
`font-weight` в декларации (variable font) — вес регулируется явным
`font-variation-settings: 'wght' N`, а не выбором другого файла.
Использование:

```css
.display-heavy { font-variation-settings: 'wght' 900; } /* хиро, вордмарк */
.display-mid   { font-variation-settings: 'wght' 700; } /* заголовки секций */
.display-label { font-variation-settings: 'wght' 500; } /* лейблы, eyebrow */
```

**Только для дисплейных моментов** — вордмарк, заголовки секций,
крупные числа/цены. Тело интерфейса остаётся на системном стеке
(`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, Helvetica,
Arial, sans-serif`) — веб-шрифт в рантайме основного текста не грузим
намеренно, это ограничение перфоманса в Telegram WebView на бюджетных
Android (см. [STACK.md#вёрстка](../STACK.md)), не вопрос тренда, и оно
не пересматривалось Y2K-поворотом.

## Приёмы

- **Хром-градиент на тексте** — только на вордмарке (`gramma` в
  `linear-gradient(180deg, --chrome-light, --chrome 45%, --chrome-dark)`
  через `background-clip: text`), не на заголовках секций — решено
  явно 19.07.2026, не расширять без нового решения.
- **Glow** — magenta-glow на CTA/цены/скидки (`text-shadow`/`box-shadow`
  с `--magenta` низкой непрозрачности), ice-glow на интерактивных/
  статусных элементах. Символ пары: сигнатурный приём эпохи, не
  использовать на инертных/неважных элементах — обесценится.
- **Скан-линии** — едва заметная фактура фона, `repeating-linear-gradient`
  горизонтальными линиями каждые 3px, непрозрачность 3.5%:
  ```css
  background: repeating-linear-gradient(
    0deg,
    rgba(184, 196, 208, 0.035) 0px,
    rgba(184, 196, 208, 0.035) 1px,
    transparent 1px,
    transparent 3px
  );
  ```
  Решено 19.07.2026 — интенсивность **не усиливать**.

## Формы

Зафиксировано в диалоге 22.07.2026 (вне спринта — концептуальное решение,
см. [tasks/milestones.md](../tasks/milestones.md)). Правила разметки и
поведения форм — не только визуал, но и валидация: раскол между экранами
(`ProductForm`/`RegisterForm`/`SellerProfile` дизейблили submit до
валидности формы, `CheckoutPage` валидировал вживую с инлайн-ошибками)
показал, что без явного правила поведение расползается так же, как и
вёрстка. Заземлено на исследовании 22.07.2026 (Baymard Institute, NN/g,
LukeW), не на памяти — см. «Источники».

**Раскладка полей**:
- Один столбец, одно поле в строке — не многоколоночные формы. Baymard:
  multi-column даёт больше пропущенных и перепутанных полей;
  CXL-измерения — single-column проходят в среднем на 15 секунд быстрее.
- Исключение — только жёстко спаренные поля с очевидной смысловой связью
  (город+индекс, дата+CVC). В текущих формах grammashop таких пар нет —
  правило декларативное, на будущее, не применяется нигде сейчас.
- Ширина поля — всегда на всю ширину контейнера формы (`w-full`), без
  вариаций по типу поля.
- Лейбл — сверху над полем (top-aligned), не сбоку: на мобильном экране
  (а именно так открывается интерфейс в Telegram Mini App) это не требует
  масштабирования и читается за одну фиксацию взгляда.

**Обязательные/необязательные поля**: в формах grammashop бо́льшая часть
полей обязательна — по актуальной рекомендации NN/g в этом случае
помечать нужно не обязательные поля звёздочкой, а меньшинство —
**необязательные**, суффиксом `(необязательно)` в тексте лейбла.
Обязательные поля — без пометки вообще.

**Валидация и ошибки** — канон: `CheckoutPage.tsx` +
`checkout/validate-checkout-form.ts`.
- Проверка поля — по `blur` (после того как пользователь ушёл с поля), не
  на каждое нажатие клавиши: ошибка, всплывающая посреди набора текста,
  выглядит как обвинение до того, как ввод закончен, и на мобильном
  сдвигает разметку под пальцем.
- Ошибка — текстом под конкретным полем (не одной общей строкой под всей
  формой), плюс подсветка рамки поля и `aria-invalid`/`aria-describedby`
  на самом инпуте.
- Кнопка submit **не дизейблится по невалидности формы** — только на
  время самого запроса (`isPending`). Дизейбл до полной валидности —
  антипаттерн: пользователь не понимает, почему кнопка не нажимается.
  По попытке сабмита с ошибками — все поля помечаются «тронутыми», ошибки
  подсвечиваются, отправка блокируется валидацией, а не состоянием
  кнопки.

**Не реализовано**: `RegisterForm.tsx`, `SellerProfile.tsx`,
`ProductForm.tsx` пока на старом паттерне (дизейбл кнопки, общая ошибка
без разбивки по полям, без пометки необязательных полей) — перевод на
этот канон в бэклоге (`docs/tasks/backlog.md`), не сделан попутно с
фиксацией правил.

## Символ и логотип

Файл: [`logo.svg`](logo.svg) — хромированная гиря (калибровочный груз:
купол-«гриб» + перехват-«талия» + цилиндрическое тело), сгенерирована
через AI-инструмент с нативным векторным рендерингом (не хендкод SVG —
см. «История»). Многотональный рендер (белый/серый/чёрный) уже
соответствует токену `--chrome` — **не перекрашивать**.

Правила использования:
- Фон в файле отсутствует (удалён при интеграции) — знак самостоятельный,
  ложится на любую поверхность. Проверено на `--void`, на `--chrome`, на
  белом.
- На светлом `--chrome`-фоне контраст слабее (белые блики частично
  сливаются с фоном) — не проблема на практике: `--chrome` в системе не
  используется как заливка под крупными элементами, только текст/обводки.
- Держится вплоть до 16px (проверено вживую рендером, не на глаз) —
  безопасно использовать как favicon.
- Не растягивать не пропорционально, не добавлять обводку/тень поверх
  уже отрендеренных бликов — испортит читаемость металла.

Лок-ап с вордмарком: знак слева, `gramma┃shop` справа, разделитель между
`gramma` и `shop` — вертикальная magenta-линия с glow (та же роль, что у
хром-градиента в вордмарке — сигнатурная деталь, не декоративная).

## Реализация

Токены и приёмы применены в коде `apps/web` Спринтом 17 (20.07.2026,
[tasks/17-y2k-storefront-design.md](../tasks/17-y2k-storefront-design.md)) — но только на витрине покупателя, не везде:
`index.css` (токены + утилити-классы), `StorefrontHome.tsx` (флюидная
сетка каталога), `ProductCard.tsx`, `ProductDetail.tsx`. `CartPage.tsx`/
`CheckoutPage.tsx` — ещё на прежней нейтральной `--tg-*`-системе без
Y2K-акцента (не входили в скоуп Спринта 17). Обе админки, лендинг —
Y2K не касается по принципу «не везде, где витрина продавцу не
принадлежит» (см. выше), это не долг, а осознанная граница.

**Находка при переводе в код**: чистый `--magenta`/`--ice` текстом
поверх переменной `--tg-surface`/`--tg-bg` (светлая тема Telegram)
проваливает контраст — эти цвета калибровались на прототипах против
`--void` (тёмная брендовая поверхность), не против произвольной темы
клиента. Решение — токен `--magenta-on-theme` (`color-mix` с
`--tg-text`, детали в `tasks/17-y2k-storefront-design.md`), `--ice` текстом на
`--tg-surface` не используется вовсе (только `--tg-link`/`--tg-text`
в этих местах). Не пересматривает саму палитру — только то, где именно
подставляется голый `--magenta`/`--ice`, а где адаптированный вариант.

**Минимальный ДНК в админках — уточнено 20.07.2026 (Спринт 19)**:
принцип «обе админки не подключают Y2K» выше — не про нулевой контакт с
брендом, а про то, что админки не получают полный акцент (сетку/фото/
glow/скан-линии/тему продавца). Ради единого дизайн-кода продукта
(запрошено в диалоге — цельность инструментов важнее строгой границы
«витрина/остальное») туда переезжают две вещи: `font-display`
(Unbounded) на заголовках экранов и `--magenta`/`--magenta-on-theme`
как акцент главной созидающей кнопки экрана («Добавить»/«Сохранить»).
Всё остальное (плотная плоская раскладка, нейтральные вторичные
действия, отсутствие glass/scan-lines) — без изменений, см.
[STACK.md#дизайн-направление](../STACK.md#дизайн-направление).

Также не сделано: favicon `apps/web/index.html` (нужен экспорт
`logo.svg` в `.ico`/набор PNG-размеров), аватарка `@grammashopbot` в
BotFather (ручная загрузка вне кода).

## История

- **Черновик 1-2 (18-19.07.2026, отклонён целиком)** — тихий минимализм
  по референсам азиатской fashion-индустрии (Markaware, 29CM, W
  Concept, Ader Error), палитра Ink/Paper/Brass, вордмарк на Fraunces
  (серифная антиква без кириллицы). Символ перебирал через несколько
  гипотез — весы (отклонено: не относится к смыслу проекта), монета с
  самолётиком Telegram (отклонено), змея вокруг пачки денег (отклонено:
  ручной SVG не держал форму), гиря/калибровочный груз хендкодом
  (отклонено: «получается шляпа» — суть формы была верной, но ручное
  написание SVG-координат — плохой инструмент для векторной
  иконографии).
- **Поворот на Y2K (19.07.2026)** — по прямому решению: «жирные прямые
  шрифты», хром вместо золота. Символ решён генерацией через AI-
  инструмент (не хендкод) — та же форма гири, что и в отклонённом
  черновике, но исполнение через нормальный инструмент вместо ручных
  bezier-координат.
